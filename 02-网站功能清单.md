# 网站功能清单

## 1. 平台第一阶段必须有的公共功能

以下能力属于平台本身，不属于某个具体模块：

- 统一登录页
- 平台首页
- 模块中心
- 全局搜索入口
- 任务中心
- 文件中心
- 操作日志中心
- 通知入口
- 系统设置入口
- 统一账号与角色基础
- 模块注册与状态展示
- 统一错误、空状态、加载状态表达
- 从任意模块返回平台的统一入口

## 2. 平台第二阶段应补齐的公共功能

- 细化权限与角色
- 通知提醒策略
- 文件分类与归档
- 平台级筛选与搜索增强
- 数据备份与恢复
- 模块使用统计
- 平台帮助与新手引导

## 3. 模块层建议具备的标准能力

平台不预设固定旧模块，但要求后续任何模块都从以下能力中组合：

### 3.1 标准工作台能力

- 模块首页或工作台
- 关键摘要
- 最近任务
- 快速操作入口

### 3.2 数据与对象管理能力

- 列表页
- 详情页
- 筛选与搜索
- 新建、编辑、删除
- 历史记录

### 3.3 导入导出能力

- 文件导入
- 导入预览与校验
- 导出结果
- 导入导出历史

### 3.4 流程与任务能力

- 发起任务
- 查看进度
- 查看结果
- 失败反馈
- 重试

### 3.5 内容与知识能力

- 文档或内容展示
- 结构化内容分类
- 收藏、标记或引用

### 3.6 轻量交互能力

- 手机端友好页面
- 简化流程入口
- 沉浸式操作页

## 4. 建议作为新平台优先考虑的高级功能

- 平台级模块注册中心
- 平台级文件服务
- 平台级任务状态模型
- 平台级日志与审计模型
- 平台级通知机制
- 平台级搜索索引
- 平台级配置管理

## 5. 第一阶段当前不要做的功能

- 一开始就做过多业务模块
- 一开始就做复杂多租户
- 一开始就做插件市场
- 一开始就做复杂消息队列和微服务拆分
- 一开始就做花哨但无实际价值的视觉层
- 没有明确使用场景的新功能

## 6. 功能优先级建议

第一优先级：

- 平台公共层成型
- 统一导航与登录
- 统一任务、文件、日志基础
- 一个标准样板模块

第二优先级：

- 搜索、通知、设置
- 第二模块形态验证
- 模块接入标准落地

第三优先级：

- 模块联动
- 权限细化
- 平台级运营与统计
- 更多业务模块
