# 模块应有功能清单

本文件用于回答一个问题：

当前这个统一平台，按照“团队内部长期使用”的标准，每个正式模块和平台公共层，应该至少具备哪些功能。

这不是“当前已实现清单”，而是“目标能力清单”。

后续所有 agent 在继续补功能、做增强、做精修前，必须先阅读：

1. `ROADMAP.md`
2. `当前真实状态.md`
3. `最终验收清单.md`
4. 本文件 `06-模块应有功能清单.md`

如果代码现状和本文件不一致，以本文件作为后续补全方向，但实现前仍要先核对真实代码状态。

---

## 1. 文档用途

本文件解决三类偏航问题：

- 只修 bug，不知道产品最终要长成什么样
- 只补页面，不补结果页、联动、权限、日志、导出
- 只做单模块，不考虑统一平台的公共能力

因此本文件按两层组织：

- 平台公共层应有能力
- 各正式模块应有能力

每项能力都按以下等级看待：

- `P0 必备`：缺失时不能称为成熟可用
- `P1 重要`：影响日常效率与完整性，建议尽快补齐
- `P2 增强`：不影响最小闭环，但会明显提升可用性和专业度

## 2. 平台公共层应有能力

统一平台不是一组页面的集合，必须先具备公共底座。

### 2.1 统一认证与会话

- `P0` 登录、退出、当前用户信息获取
- `P0` 权限列表回读与前端入口按权限控制
- `P0` Cookie / Session 鉴权在 HTTP 与 WebSocket 上行为一致
- `P1` 会话过期提示与失效后自动回登录页
- `P1` 多角色可切换测试账号或角色验证方式
- `P2` 登录审计、异常登录告警、设备会话管理

### 2.2 统一导航与模块注册

- `P0` 模块中心展示正式模块列表、状态、描述、入口
- `P0` 模块 key、前端路由、后端 API 前缀统一
- `P0` 首页、模块中心、模块内 Tab 三层导航一致
- `P1` 模块状态区分：可用、建设中、停用
- `P1` 模块入口支持按角色显示
- `P2` 模块收藏、最近访问、常用模块排序

### 2.3 统一任务中心

- `P0` 所有耗时操作进入统一任务中心
- `P0` 任务具备状态、类型、来源模块、发起人、创建时间、结果摘要
- `P0` 任务列表支持搜索、筛选、分页
- `P0` 任务详情可回到原模块对象
- `P1` 任务失败原因可见
- `P1` 任务支持取消、重试、再次执行
- `P2` 任务支持批量操作、标签、优先级、执行耗时统计

### 2.4 统一文件中心

- `P0` 模块上传的文件统一纳入文件中心
- `P0` 文件具备名称、原始名、大小、类型、所属模块、上传人、时间
- `P0` 文件列表支持搜索、筛选、分页
- `P0` 文件支持下载、归档、查看来源模块
- `P1` 文件预览能力，至少覆盖 txt/json/csv/image/pdf 常见类型
- `P1` 文件与任务、记录对象的关联展示
- `P2` 文件版本、重复文件识别、批量归档/删除

### 2.5 统一日志中心

- `P0` 所有关键操作写入统一操作日志
- `P0` 日志包含动作、对象类型、对象 ID、模块、操作人、结果、时间
- `P0` 日志支持搜索、筛选、分页
- `P1` 日志支持查看 detail 明细
- `P1` 日志与任务、对象、通知可互相跳转
- `P2` 日志导出、审计报表、异常操作聚合

### 2.6 统一通知中心

- `P0` 关键事件触发站内通知
- `P0` 通知支持已读、全部已读
- `P0` 通知能跳回原业务对象
- `P1` 按类型筛选通知，如成功、失败、提醒、系统消息
- `P1` WebSocket 实时通知
- `P2` 邮件/Webhook/IM 外发通知策略

### 2.7 统一搜索

- `P0` 跨任务、文件、日志、模块、记录对象搜索
- `P0` 返回结果有类型标记与跳转链接
- `P1` 支持按模块、类型过滤
- `P1` 搜索结果高亮命中内容
- `P2` 保存搜索、最近搜索、组合筛选

### 2.8 统一统计与首页

- `P0` 首页读取真实平台统计，不展示硬编码假数据
- `P0` 展示任务、文件、记录对象、活跃模块等核心指标
- `P0` 展示最近任务或最近活动
- `P1` 不同角色可看到不同首页重点
- `P1` 支持时间范围切换
- `P2` 支持自定义看板组件

### 2.9 统一对象联动

- `P0` 各模块关键对象可相互跳转
- `P0` 至少具备“来源”“关联对象”“回到原模块”的联动链路
- `P0` 跨模块关联动作写日志
- `P1` 记录对象作为统一连接点
- `P2` 关联图谱、关联时间线

### 2.10 统一导入导出与备份

- `P0` 平台具备备份能力与下载入口
- `P1` 模块数据支持导出，至少覆盖 json/csv/txt 中的适合格式
- `P1` 关键对象导入有结果反馈
- `P2` 增量备份、恢复演练、导入校验报告

### 2.11 统一帮助与设置

- `P0` 新用户知道平台能做什么、每个模块做什么
- `P0` 设置页能看到角色、权限、系统能力概况
- `P1` 帮助页按模块提供使用说明
- `P1` 常见问题与错误提示可查
- `P2` 操作引导、产品更新记录

### 2.12 统一质量门

- `P0` 前端 `lint` 通过
- `P0` 前端 `build` 通过
- `P0` 后端测试通过
- `P0` 前端冒烟测试通过
- `P1` 每个正式模块至少 1 条真实闭环测试
- `P2` 回归测试按模块扩展

## 3. 分析工作模块应有能力

模块 key：`analysis-workbench`

### 3.1 核心闭环

- `P0` 新建分析任务
- `P0` 上传分析文件
- `P0` 发起分析
- `P0` 查看分析结果
- `P0` 查看历史任务
- `P0` 导出分析报告

### 3.2 任务与结果能力

- `P0` 任务状态完整：待执行、执行中、成功、失败、取消
- `P0` 分析结果有摘要，而不是只有“完成”
- `P0` 结果能生成关联记录对象
- `P0` 结果进入任务、日志、通知链路
- `P1` 可查看输入文件清单
- `P1` 可查看输出结果项明细
- `P1` 支持按任务重跑
- `P2` 支持模板化分析配置、参数复用

### 3.3 文件与报告

- `P0` 上传文件状态可见
- `P0` 无文件时禁止分析并给出明确提示
- `P1` 支持多文件、混合类型文件
- `P1` 报告导出至少覆盖 txt/json
- `P2` 报告支持更完整的结构化版式，如 markdown/pdf

### 3.4 联动

- `P0` 分析结果可跳到对象管理模块查看关联对象
- `P0` 分析完成触发通知
- `P1` 结果回写搜索索引
- `P2` 支持把分析结果直接转成后续调度规则或任务

- `P0` 查看对象详情
- `P0` 编辑对象

### 4.2 列表能力

- `P0` 支持按关键词搜索
- `P0` 支持按分类、状态、标签筛选
- `P0` 支持分页
- `P1` 支持排序
- `P1` 支持批量操作
- `P2` 支持保存筛选方案

### 4.3 详情能力

- `P0` 展示基础字段、标签、状态、负责人、时间

- `P0` 可回看关联日志
- `P1` 可查看来源分析批次或关联任务

## 9. 当前建议的补全优先级

如果后续继续推进，建议按以下顺序补，而不是平均用力。

### 第一优先级

- 平台公共层剩余统一能力补全
- 分析模块结果细化与重跑能力
- 对象管理模块的详情联动与批量操作
- 调度模块执行历史与失败处理

### 第二优先级

- 学习训练模块的继续练习、错题复练、训练内容扩展
- Dashboard 的时间维度统计与角色化展示
- 文件预览、日志详情、通知类型筛选

### 第三优先级

- 自定义看板
- 更强的导出能力
- 更专业的分析模板和规则引擎
- 更强的搜索与关联图谱

## 10. 功能完整度判断标准

后续 agent 补功能时，不允许只看“页面有没有”，而要按以下标准判断某项功能是否算完成。

一项功能只有同时满足以下条件，才可视为“已完成”：

1. 有入口
2. 有真实后端能力
3. 有成功路径
4. 有失败提示
5. 有结果承接页或结果状态
6. 能进入日志、任务、通知、联动中的至少一部分
7. 有自动化测试或人工验收记录

如果只是：

- 有按钮但没真实效果
- 有接口但前端没入口
- 有页面但数据是写死的
- 有成功提示但无结果页

都不能算完成。

## 11. 后续执行要求

后续任何 agent 如果要补模块功能，必须先做三件事：

1. 对照本文件确认目标能力
2. 对照 `当前真实状态.md` 确认真实现现状
3. 明确这次修改属于 `P0 / P1 / P2` 哪一层

禁止直接说“继续完善一下”而不指明是在补哪一类能力。
