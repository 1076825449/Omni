# 变更记录规范

## 1. 为什么必须做变更记录

这个平台后期一定会继续增长：

- 会增加新模块
- 会给已有模块加功能
- 也会删掉一些旧功能

如果没有变更记录，几轮 AI 开发后就会出现：

- 你不知道现在到底有哪些功能
- 你不知道哪些是后来加的
- 你不知道哪些旧功能被删了
- 你不知道某个页面为什么改成现在这样

所以变更记录必须长期维护。

## 2. 每次变更至少记录什么

每次新增、修改、删除，都至少要记录：

- 变更日期
- 变更名称
- 变更类型：新增 / 修改 / 删除
- 所属层级：平台 / 模块
- 影响范围
- 变更原因
- 结果说明
- 验收结论

## 3. 建议记录方式

建议单独维护一份持续更新的变更总表，按时间排序。

每条记录建议写成：

- 日期
- 标题
- 类型
- 涉及层级或模块
- 简述
- 是否已验收

## 4. 删除类变更必须额外记录

如果是删除旧功能，必须多写两项：

- 替代方案
- 是否影响旧用户习惯

## 5. 版本意识

即使你不做严格的软件版本管理，也建议保留“阶段版本”概念，例如：

- 平台重建 V1
- 平台稳定版 V1.1
- 平台扩展版 V1.2

这样更容易判断当前平台处在哪个阶段。
