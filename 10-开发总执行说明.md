# 开发总执行说明

## 1. 目的

这份文档用于把后续开发变成一条连续可执行的任务链，而不是零散开发。

要求：

- AI 必须按任务顺序执行
- 一个任务验收通过后，才能进入下一个任务
- 每个任务完成后都必须提交 Git
- 每个任务都必须产出验收结果

## 2. 总执行顺序

后续开发统一按以下顺序推进：

1. `20-任务包-00-总执行规则.md`
2. `21-任务包-01-项目初始化与文档落位.md`
3. `22-任务包-02-平台信息架构与页面清单.md`
4. `23-任务包-03-平台视觉基线与设计系统.md`
5. `24-任务包-04-平台骨架前端实现.md`
6. `25-任务包-05-登录与会话基础.md`
7. `26-任务包-06-模块中心与导航.md`
8. `27-任务包-07-统一任务记录与日志骨架.md`
9. `28-任务包-08-标准模块样板.md`
10. `29-任务包-09-平台公共高级能力.md`
11. `30-任务包-10-第二模块形态验证.md`
12. `31-任务包-11-平台第一阶段验收与交付.md`

## 3. 执行原则

- 不允许跳任务
- 不允许同时大范围改多个任务
- 不允许一个任务没验收就开始下一个
- 不允许未提交 Git 就继续后续任务

## 4. 每个任务的完成定义

一个任务只有同时满足以下条件才算完成：

- 功能完成
- 符合平台规范
- 验收说明已给出
- 风险与未完成项已说明
- Git 已提交

## 5. AI 每次开始任务前必须读取

- `README.md`
- `01-网站产品定义.md`
- `02-网站功能清单.md`
- `03-统一平台总规范.md`
- `04-模块设计规范.md`
- `05-AI开发协作规范.md`
- `11-Git提交规范.md`
- 当前任务包
- `07-验收清单模板.md`

## 6. 阶段边界

### 第一阶段：平台骨架

- 任务 01 到 07

### 第二阶段：核心模块接入

- 任务 08 到 10

### 第三阶段：整体验收收口

- 任务 11

## 7. 当前建议

你后续给 AI 下指令时，直接说：

“请严格按 `10-开发总执行说明.md` 与当前任务包执行。当前只做第 X 个任务，不得提前进入下一个任务。完成后必须 Git 提交并按验收模板输出结果。”
