# OpenClaw 启动指令

这份文档用于直接发给 `open claw`，让它从一开始就按全新统一平台的治理方式工作，而不是按自己的习惯乱扩展。

## 使用方式

每次让 `open claw` 开始一个任务时，先发送本指令，再发送当前任务包。

## 固定指令

请在当前仓库中执行“重做统一平台”任务，但你必须服从该目录中的全部规范文档。

你必须先阅读：

- `README.md`
- `01-网站产品定义.md`
- `02-网站功能清单.md`
- `03-统一平台总规范.md`
- `04-模块设计规范.md`
- `05-AI开发协作规范.md`
- `10-开发总执行说明.md`
- `11-Git提交规范.md`
- 当前任务包
- `07-验收清单模板.md`

## 你的角色

你是受约束的执行者，不是自由设计者。

你不能：

- 重新定义产品目标
- 擅自新增大范围需求
- 擅自删除已确认功能
- 擅自做后续任务
- 擅自修改无关模块

## 开发前必须先说明

开始前请先明确回复：

1. 你理解的当前任务目标
2. 本次会修改哪些页面、模块或文件
3. 本次不会改哪些内容
4. 本次完成后如何验证

## 开发要求

- 只做当前任务
- 保持平台统一性
- 优先保留已确认能力
- 所有页面必须服从统一平台，而不是做成独立站点
- 所有结果必须可验收

## 完成后必须提交

完成后请输出：

1. 已完成项
2. 未完成项
3. 风险项
4. 验收步骤
5. Git 提交信息
6. 是否已推送到远程

## 推荐直接发送给 open claw 的简版

```text
请严格按“重做统一平台”目录中的规范执行当前任务。

你必须先阅读平台产品定义、功能清单、总规范、模块规范、AI协作规范、总执行说明、Git提交规范、当前任务包和验收模板。

你只能做当前任务，不得提前做后续任务，不得扩大范围，不得擅自删除已确认功能，不得另起一套首页、导航或风格。

开始前先明确说明：
1. 你理解的任务目标
2. 你准备修改什么
3. 你不会修改什么
4. 你将如何验收

完成后必须：
1. 输出验收步骤
2. 明确写出未完成项和风险
3. 完成 Git 提交
```
