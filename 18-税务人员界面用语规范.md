# 税务人员界面用语规范

## 1. 目标

普通业务页面必须使用税务人员能直接理解的工作语言，不把工程术语暴露给一线用户。

## 2. 模块展示名称

1. `analysis-workbench` 展示为 `案头分析`
2. `info-query` 展示为 `纳税人信息查询`
3. `risk-ledger` 展示为 `风险记录台账`
4. `dashboard-workbench` 展示为 `工作统计`
5. `schedule-workbench` 展示为 `自动任务`
6. `record-operations` 展示为 `辅助数据管理`，不作为税源管理员主入口

## 3. 常用动作

1. `新增`：优先写成 `新增风险记录`、`新增整改记录`
2. `同步`：优先写成 `记入风险台账`
3. `对象`：面向普通用户时改为 `风险事项`、`企业记录` 或 `工作记录`
4. `任务`：按场景写成 `案头分析任务`、`整改跟踪事项`
5. `导出`：明确写成 `导出分析报告`、`生成税务事项通知书`
6. `批量处理`：按业务结果写成 `批量标记已排除`、`批量标记整改中`、`批量标记已整改`
7. `下载`：清单类资料优先写成 `导出清单`
8. `截止日期`：风险整改场景统一写成 `整改期限`
9. `负责人`：面向企业联系场景统一写成 `联系人`、`联系电话`

## 4. 状态

风险处理状态固定使用：

1. `待核实`
2. `已排除`
3. `整改中`
4. `已整改`

登记状态来自纳税人信息表，必须与风险处理状态分开展示。

## 5. 禁止直接暴露

普通业务页面不应直接出现以下词语：

1. `module key`
2. `API`
3. `Dashboard`
4. `Schedule`
5. `对象ID`
6. `batch`
7. `record_id`

如确需保留，应仅在管理员、日志、调试或系统设置页面展示。
