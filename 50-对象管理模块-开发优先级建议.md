# 对象管理模块开发优先级建议

## 1. 目的

这份文档用于约束另一个 agent 在开发对象管理模块时，优先把高频、刚需、批量操作类能力做顺，而不是先做复杂规则系统。

核心原则：

- 先把“列表 -> 筛选 -> 详情 -> 批量处理 -> 导出”做成顺手的管理闭环

## 2. 本模块的主动作

对象管理模块的主动作是：

- 管理一批对象，并对它们进行分类、分配、批量调整和导出

## 3. 第一阶段必须有

以下能力必须先做出来：

- 模块工作台
- 导入页
- 对象列表页
- 条件筛选
- 对象详情页
- 分类处理
- 分配处理
- 单条编辑
- 批量操作
- 导出结果
- 导出记录
- 操作日志

## 4. 第一阶段必须做好

### 4.1 列表必须顺手

列表页必须做到：

- 筛选清楚
- 搜索清楚
- 批量选择清楚
- 操作入口清楚

### 4.2 详情必须能解释结果

详情页不能只显示最终结果。  
必须尽量告诉用户：

- 为什么会被分到这个结果
- 它当前属于什么状态
- 最近做过哪些调整

### 4.3 批量操作必须有确认提示

批量操作前必须提示：

- 将影响多少条记录
- 将执行什么操作
- 是否可撤销或如何修正

### 4.4 导出必须可追踪

导出不应只是一个按钮。  
至少要有：

- 导出记录
- 最近导出结果
- 导出状态

## 5. 第二阶段再做

这些内容可以后做：

- 更复杂的规则引擎编辑器
- 更复杂的推荐分配
- 更复杂的批量审批逻辑
- 更复杂的多层级对象关系
- 更复杂的跨模块自动流转

## 6. 当前先不要做

以下内容第一阶段明确不要做：

- 可视化拖拽规则系统
- 超复杂权限树
- 自动跨模块同步
- 大量智能推荐逻辑
- 复杂组织结构治理

## 7. 页面建议

优先级建议：

1. 工作台页
2. 导入页
3. 列表页
4. 详情页
5. 批量操作页
6. 导出页
7. 日志页
8. 规则配置页

## 8. 验收重点

验收对象管理模块时，重点看：

- 导入是否清楚
- 列表是否顺手
- 详情是否解释得通
- 批量操作是否安全
- 导出是否可追踪
- 日志是否完整

## 9. 对另一个 agent 的直接约束

开发本模块时：

- 不要先做复杂规则系统
- 不要先做复杂权限结构
- 不要让列表页操作变得过重
- 不要让详情页只给结果不给原因

一句话：

先把管理闭环做顺，再做复杂规则。
