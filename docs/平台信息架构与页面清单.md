## 1. 整体架构

```
Omni 统一平台
├── 平台公共层
│   ├── 登录 / 会话
│   ├── 首页
│   ├── 导航
│   ├── 模块中心
│   ├── 任务中心
│   ├── 文件中心
│   ├── 日志中心
│   ├── 通知中心
│   └── 系统设置
└── 业务模块层（按模块独立）
    ├── 分析工作模块 /modules/analysis-workbench
    ├── 对象管理模块 /modules/record-operations
    └── 学习训练模块 /modules/learning-lab
```

**核心原则：**
- 平台只有一套总导航，模块内部不再重建总导航
- 模块只能有模块级侧边/顶部导航
- 从任意模块必须能返回平台首页

## 3. 导航结构

### 3.1 顶部固定导航栏（所有页面可见）

```
[Logo/平台名]  [首页] [模块中心] [任务中心] [文件中心] [日志中心]     [通知🔔] [头像/设置⚙️]
```

- 左侧：平台 Logo + 平台名称（点击回首页）
- 中间：全局导航（首页、模块中心、任务、文件、日志）
- 右侧：通知图标 + 用户头像/设置

### 3.2 模块内部导航

模块内部只允许出现**模块级子导航**，不允许重建平台总导航。

模块子导航放在左侧或顶部 Tab，以模块名作为一级类目。

```
模块名：分析工作模块
子导航：[工作台] [新建] [历史] [报告]
```

## 6. 用户核心流程

```
1. 访问 /login
   ↓ 登录成功
2. 平台首页 /
   ↓
3. 可选：进入模块中心 /modules，选择模块
   ↓
4. 进入具体模块页面 /modules/<module-key>
   ↓
5. 在模块内操作（任务/文件/日志）
   ↓
6. 从任意模块 → 点击顶部 Logo 或"首页" → 返回平台首页
```

## 7. 统一状态设计（所有页面必须遵循）

每个页面必须统一处理以下状态：

| 状态 | 用户看到 |
|------|---------|
| 加载中 | 骨架屏 / 加载动画 |
| 空数据 | 空状态插画 + 引导文案 |
| 操作成功 | 成功提示 + 结果展示 |
| 操作失败 | 错误说明 + 可行的下一步 |
| 权限不足 | 权限提示 + 联系管理员 |
| 任务进行中 | 进度条 + 任务状态 |
| 处理中 | 按钮 loading + 禁用状态 |

## 8. 平台与模块边界规则

| 能力 | 归属 |
|------|------|
| 登录/认证 | 平台公共层 |
| 首页/导航 | 平台公共层 |
| 模块注册/展示 | 平台公共层 |
| 任务中心/文件中心/日志中心 | 平台公共层 |
| 通知/设置 | 平台公共层 |
| 模块具体业务页面 | 各模块自己 |
| 模块内部任务/文件/日志 | 各模块，但统一接入平台模型 |
