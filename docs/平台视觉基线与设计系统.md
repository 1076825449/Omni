# 平台视觉基线与设计系统

> 说明：本文件是早期视觉规范稿，提供设计原则参考，不代表当前前端实现已逐项完全落地。当前执行与验收基线请以仓库根目录的 `ROADMAP.md`、`最终验收清单.md`、`当前真实状态.md`、`最终验收结论.md` 为准。

> 任务包 03 产出物 | 平台公共层 | 版本：v0.1.0

---

## 1. 视觉方向

**关键词：清晰、稳重、专业、可信赖**

视觉基调参考 Ant Design 的企业级设计语言，但经过平台层二次封装，避免裸用 Ant Design 原生样式造成风格漂移。

核心原则：
- 以内容为主，不以视觉为主
- 间距和层次优先于装饰
- 风格服务于效率，不服务于炫技
- 移动端友好，响应式优先

## 2. 设计 tokens

### 2.1 颜色

| token | 色值 | 用途 |
|-------|------|------|
| `--omni-color-primary` | `#1677ff` | 主色：按钮、链接、选中 |
| `--omni-color-primary-hover` | `#4096ff` | 主色悬停 |
| `--omni-color-primary-active` | `#0958d9` | 主色按下 |
| `--omni-bg-base` | `#ffffff` | 页面底色 |
| `--omni-bg-layout` | `#f5f7fa` | 布局底色 |
| `--omni-bg-card` | `#ffffff` | 卡片底色 |
| `--omni-bg-hover` | `#f0f5ff` | 悬停背景 |
| `--omni-text-primary` | `#1d2129` | 主要文字 |
| `--omni-text-secondary` | `#4b5563` | 次要文字 |
| `--omni-text-tertiary` | `#86909c` | 占位/提示文字 |
| `--omni-border-color` | `#e5e6eb` | 边框色 |

### 2.2 字体

- 主字体：`system-ui, -apple-system, 'Segoe UI', Roboto, 'Helvetica Neue', Arial, sans-serif`
- 等宽字体：`'SF Mono', 'Fira Code', Consolas, monospace`（用于代码/Key 显示）
- 基础字号：14px
- 行高：1.6

### 2.3 间距

| token | 值 | 用途 |
|-------|---|------|
| `--omni-spacing-xs` | 4px | 紧凑间距 |
| `--omni-spacing-sm` | 8px | 小间距 |
| `--omni-spacing-md` | 16px | 标准间距 |
| `--omni-spacing-lg` | 24px | 大间距 |
| `--omni-spacing-xl` | 32px | 区块间距 |

### 2.4 圆角

| token | 值 | 用途 |
|-------|---|------|
| `--omni-border-radius-sm` | 6px | 按钮/输入框 |
| `--omni-border-radius` | 8px | 卡片/面板 |
| `--omni-border-radius-lg` | 12px | 大容器 |

### 2.5 阴影

| token | 值 | 用途 |
|-------|---|------|
| `--omni-shadow-1` | `0 1px 2px rgba(0,0,0,0.04)` | 默认卡片 |
| `--omni-shadow-2` | `0 2px 8px rgba(0,0,0,0.08)` | 悬停卡片 |
| `--omni-shadow-3` | `0 4px 16px rgba(0,0,0,0.12)` | 弹窗/登录框 |

## 3. 布局规范

### 3.1 页面基础布局

```
┌─────────────────────────────────────────┐
│  PlatformHeader（固定顶部，高度 56px）    │
├─────────────────────────────────────────┤
│                                         │
│           页面内容区                     │
│                                         │
└─────────────────────────────────────────┘
```

- Header 固定在顶部，不随页面滚动
- 内容区最大宽度：1200px，水平居中
- 内容区内边距：24px

### 3.2 页面通用结构

每个页面必须包含：

```jsx
<div className="omni-page">
  <div className="omni-page-header">
    <h1 className="omni-page-title">页面标题</h1>
    <p className="omni-page-subtitle">副标题/说明</p>
  </div>
  <main>
    {/* 页面内容 */}
  </main>
</div>
```

### 3.3 栅格系统

使用 Ant Design 的 Row/Col 栅格，参照以下断点：
- `xs`：< 576px（手机）
- `sm`：≥ 576px（平板）
- `md`：≥ 768px
- `lg`：≥ 992px
- `xl`：≥ 1200px

## 4. 平台头部导航

### 4.1 组成

```
[Logo/平台名]  [首页] [模块中心] [任务中心] [文件中心] [日志中心]     [通知🔔] [头像]
```

- Logo 区：左侧，点击回首页
- 导航项：居中，当前页高亮
- 右侧：通知图标 + 用户头像

### 4.2 状态

| 状态 | 样式 |
|------|------|
| 默认 | 文字色 `#4b5563`，无背景 |
| 悬停 | 背景 `#f0f5ff` |
| 选中 | 背景 `#e6f0ff`，文字 `#1677ff` |

## 5. 统一组件规范

### 5.1 卡片（Omni Card）

```css
background: #ffffff
border-radius: 8px
box-shadow: 0 1px 2px rgba(0,0,0,0.04)
border: 1px solid #e5e6eb
padding: 24px
```

悬停效果（可选）：
```css
box-shadow: 0 2px 8px rgba(0,0,0,0.08)
border-color: #1677ff
```

### 5.2 按钮

- 主按钮：`background: #1677ff`，白色文字
- 次按钮：白色背景，`border: 1px solid #e5e6eb`
- 危险按钮：`background: #ff4d4f`
- 圆角：`6px`
- 高度：标准 36px，large 40px，small 28px

### 5.3 表单

- Label 在输入框上方
- 输入框高度 36px
- 必填标记：红色 `*`
- 错误提示在输入框下方

### 5.4 表格

- 表头：`background: #fafafa`，文字加粗
- 行悬停：`background: #f0f5ff`
- 边框：横向分隔线为主，不做全部边框
- 分页：右下角

### 5.5 空状态

必须包含：
- 空状态图标/插图
- 说明文字（如"暂无数据"）
- 引导操作（如"点击新建"）

### 5.6 错误状态

必须包含：
- 错误说明（不只报 500/404）
- 用户可行的下一步操作

### 5.7 加载状态

- 骨架屏优先于 spinner
- 表格加载时显示骨架行
- 按钮 loading 时禁用并显示 spinner

## 6. 登录页规范

- 全屏背景：渐变色（`#667eea → #764ba2`）
- 登录框居中：白色卡片，圆角 12px，阴影
- 包含平台 Logo/名称
- 包含用户名 + 密码表单
- 包含"记住登录"可选

---

## 7. 模块页面规范

### 7.1 模块页面组成

```
[平台 Header]
  [模块页面标题栏]  ← 模块名 + 模块级 Tab 导航
    [模块内容区]
```

### 7.2 模块级 Tab 导航（左侧或顶部）

示例（分析工作模块）：
```
分析工作模块
[工作台] [新建分析] [历史任务] [报告]
```

### 7.3 模块内不允许出现

- 平台级总导航
- 重新定义的 Footer
- 与平台风格完全不同的配色

## 9. 后续开发必须遵守

1. 所有页面颜色必须引用 CSS 变量，不允许写死色值
2. 所有页面间距必须引用 spacing token
3. 所有页面圆角必须引用 border-radius token
4. 所有新增页面必须包含 `omni-page` 结构
5. 所有状态（空/错误/加载）必须统一处理
6. 不得在模块内单独引入另一套 UI 框架
7. 登录页必须独立于平台布局之外
