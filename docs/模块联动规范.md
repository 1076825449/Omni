# 模块联动规范 v1.0

## 1. 核心原则

- **模块自治**：各模块独立数据存储，不得直接读写对方模型表
- **平台桥接**：模块间对象引用通过平台统一 Record Link 实现
- **单向导航**：模块 A → 模块 B 的联动，单向可跳转，不强制耦合
- **日志留存**：所有跨模块跳转行为写入 `cross_link_logs`

## 2. 可共享对象范围

| 对象类型 | 说明 | 共享方式 |
|----------|------|----------|
| 用户 (User) | 统一用户体系 | 直接引用 user_id |
| 文件 (FileRecord) | 统一文件中心 | 直接引用 file_id |
| 任务 (Task) | 统一任务中心 | 直接引用 task_id |
| 日志 (OperationLog) | 统一日志中心 | 直接引用 log_id |
| 业务对象 (Record) | 跨模块业务实体 | 通过 record_id 跳转 |

## 3. 模块间跳转规则

### 规则定义
```
来源模块 → 目标模块 → 跳转参数

analysis-workbench → record-operations
  当分析结果包含对象数据时，创建关联 Record
  URL: /modules/record-operations/{record_id}

record-operations → analysis-workbench
  可从对象详情跳转至分析工作台发起分析
  URL: /modules/analysis-workbench/{task_id}

任何模块 → 日志中心
  日志列表按 module 和 operator 筛选
  URL: /logs?module={module_key}
```

### 跳转格式
- 模块首页：`/modules/{module_key}`
- 模块详情：`/modules/{module_key}/{object_id}`
- 平台列表：`/{resource_center}` (tasks / files / logs)

## 4. 联动日志

所有跨模块跳转行为记录：
```python
CrossLinkLog {
  source_module: str
  source_type: str      # "task" / "record" / "file"
  source_id: str
  target_module: str
  target_type: str
  target_id: str
  operator_id: int
  created_at: datetime
}
```

## 5. 第一版联动示例

**场景：分析工作台 → 对象管理**

1. 用户在分析工作台创建分析任务
2. 任务完成后，分析结果中的结构化对象自动同步到对象管理模块
3. 对象管理模块中创建的 Record，标记 `import_batch` 含 `analysis-{task_id}` 前缀
4. 分析任务详情页显示「相关对象」链接，点击跳转对象管理列表页

**实现方式：**
- 分析任务完成时，解析 `result_summary` 中的结构化数据
- 如含分类对象信息，创建对应 Record（category=分析模块名, import_batch=analysis-{task_id}）
- 对象管理列表页支持按 `import_batch` 筛选：`?batch=analysis-{task_id}`
- 分析任务详情页显示相关对象数量和跳转链接
