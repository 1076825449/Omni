# Omni v1.0 产品体验审查报告

审查视角：非技术普通用户、小团队日常办公人员

审查时间：v1.0 验收后，产品化改造启动前

---

## 一、当前系统适合哪些用户

**适合的用户**：
- 有一定技术理解力（知道任务、文件、日志概念）
- 能读懂中文提示
- 需要处理税务分析、对象管理、信息查询的业务人员

**不适合的用户**：
- 完全不懂"模块"、"任务"、"对象"概念的小白
- 不理解 Cron 表达式的普通办公人员
- 第一次看到系统不知道该点哪里的随机访客

---

## 二、首页审查

### 问题

1. **欢迎语太泛**：显示"欢迎回来 | 这是您的统一工作平台"——用户不知道"统一工作平台"是什么意思
2. **没有角色说明**：用户不知道自己是什么角色、能做什么
3. **没有新手引导入口**：第一次来的用户不知道从哪开始
4. **统计数字没有解释**：任务成功率 85% 是什么意思？用户该关注什么？
5. **快捷操作太少**：只有4个快捷操作，且"查看全部任务"不在这4个里
6. **没有今日概览**：用户不知道今天有没有待办

---

## 三、模块中心审查

### 问题

1. **模块名称技术化**：
   - `analysis-workbench` → 显示"分析工作模块"，用户不知道"workbench"是什么意思
   - `schedule-workbench` → "定时调度"，用户可能不理解"调度"是什么
2. **卡片缺少用户场景说明**：只显示类型（workflow/list/dashboard），不告诉用户"什么时候用这个"
3. **显示技术 key**：`Key: analysis-workbench` 暴露给普通用户，他们不需要知道
4. **没有数据概况**：用户不知道每个模块有没有数据、有多少数据
5. **没有推荐操作**：用户点进去不知道第一步做什么

---

## 四、导航审查

### 问题

1. **导航分组不清晰**：Platform/Modules/PlatformCenter 三个分组对普通用户没有明确业务含义
2. **任务中心/文件中心/日志中心/通知中心** 没有解释，这是技术术语，不是业务语言
3. **Stats** 叫"统计"，用户不知道里面有什么
4. **当前页面高亮不够明显**

---

## 五、全局空状态审查

主要页面空状态情况：

| 页面 | 当前空状态 | 问题 |
|------|-----------|------|
| 任务中心 | "暂无任务记录" | 没有告诉用户"可以做什么" |
| 文件中心 | "暂无文件" | 没有引导下一步 |
| 日志中心 | "暂无日志" | 没有引导 |
| 通知中心 | "暂无通知" | 正常但可以更好 |
| 分析模块 | 无空状态引导 | 新用户不知道如何开始 |
| 对象管理 | "当前还没有对象记录" | 稍微好一点，但没有说"对象可以来自哪里" |
| 学习训练 | 无空状态 | 新用户不知道从哪开始练习 |

---

## 六、全局错误提示审查

| 问题 | 位置 | 说明 |
|------|------|------|
| 技术 key 暴露 | 模块中心 | Key: analysis-workbench |
| 技术字段暴露 | 多处 | module、task_id、status 等英文 |
| 错误码暴露 | 待查 | HTTP 403/500 等 |
| "undefined" | 待查 | JS 原始错误 |
| 英文保留 | 多处 | Loading、Submit、Cancel 等残留 |

## 七、7个模块逐个审查

### 1. analysis-workbench（分析工作模块）

**当前名称**：分析工作模块
**用户能理解吗**：基本能，但"工作台"概念陌生

**问题**：
- 新建任务没有步骤条，用户不知道有几步
- 上传区说明复杂，用户不知道"销项发票"文件应该长什么样
- 图片/PDF 说明不够明显，用户可能误以为会自动 OCR
- 结果页术语专业：`证据`、`判断规则`、`核查方向`——普通用户可能看不懂
- 风险状态只有英文/缩写：待核实/已确认/误报/已整改/已移交——需要解释
- "导出 JSON 数据"按钮对普通用户无意义，应该隐藏或标注"仅技术人员"

**空状态**：基本没有

### 2. record-operations（对象管理模块）

**当前名称**：对象管理模块
**用户能理解吗**：有点抽象，"对象"是什么？

**问题**：
- 没有解释"对象"是什么
- 列表字段全是英文翻译：name/classification/status/owner——但界面已翻译，细节待查
- 详情页分区清楚但"来源信息"等术语需要解释
- 批量操作二次确认不够明显
- 归档说明"归档后不会删除数据"——好，但可以更清晰

### 3. info-query（纳税人信息查询）

**当前名称**：信息查询表
**用户能理解吗**：比较清晰，"纳税人信息"是业务概念

**问题**：
- 没有说明这是"基础信息库"，是其他模块的数据来源
- 导入字段说明专业："法定代表人"、"管理分局"——但这些是业务术语，应该保留同时加解释
- 企业画像页需要检查是否清晰

### 4. risk-ledger（风险记录台账）

**当前名称**：风险记录台账
**用户能理解吗**：基本能，"台账"有点专业但可接受

**问题**：
- 状态解释不够：
  - "待核实"：刚发现，尚未确认
  - "整改中"：已通知或正在处理
  - 需要更清晰的业务说明
- 时间线展示需要检查
- "临时纳税人"需要解释

### 5. learning-lab（学习训练模块）

**当前名称**：学习训练模块
**用户能理解吗**：清晰

**问题**：
- 训练集卡片需要显示难度和题量
- 答题页要清楚"第几题/共多少题"
- 结果页"错题列表"要能直接点击重练

### 6. dashboard-workbench（数据仪表盘）

**当前名称**：数据仪表盘
**用户能理解吗**：有点技术，"仪表盘"比喻需要理解

**问题**：
- 没有说明"这是给管理者看的整体情况"
- 统计指标需要解释：
  - "活跃模块"是什么意思？
  - "成功率"是什么的成功率？
- 没有新手引导，不知道看什么

### 7. schedule-workbench（定时任务）

**当前名称**：定时调度
**用户能理解吗**：基本能

**问题**：
- Cron 表达式对普通用户是完全陌生的
- "每天 9:00"比 "0 9 * * *" 好懂得多——当前已有模板按钮，好
- 但 Cron 字段没有解释，用户不知道选哪个
- 手动执行说明需要更清楚

## 八、权限体验审查

**问题**：
- viewer 角色点了按钮才发现没权限——应该在按钮阶段就隐藏或禁用
- 403 错误提示需要统一用户化
- admin/user/viewer 角色说明需要让用户看得懂

---

## 九、帮助中心审查

**问题**：
- 新手起步页面存在，流程清晰，好
- 但还需要：3分钟上手指南、常见问题（FAQ）
- 页面内提示不够——很多按钮没有 tooltip

---

## 十、视觉统一审查（快速扫视）

**问题**：
- 卡片标题层级不统一
- 空状态样式不统一
- 错误状态样式不统一
- 按钮颜色语义不统一
- 加载状态不统一

---

## 十一、优先级定义

### P0（必须改，影响核心使用）

1. 首页改造：增加欢迎语、角色说明、新手引导、快速开始、今日概览
2. 模块中心改造：去掉技术 key、增加用户场景说明、增加数据概况
3. 7个模块首页改造：增加说明、推荐操作、空状态引导
4. 全局错误提示改造：消除技术错误码，替换为用户友好提示
5. 全局空状态改造：统一空状态格式，包含"为什么没有"+"下一步做什么"
6. 导航改造：分组清晰化、命名用户化、当前页高亮

### P1（重要，影响易用性）

1. Schedule 模块 Cron 字段解释加强
2. 帮助中心完善（FAQ、3分钟上手）
3. viewer 角色按钮隐藏/禁用
4. 风险状态中文解释
5. 分析结果页术语简化

### P2（增强，体验更好）

1. 视觉统一（卡片/表格/标签/按钮）
2. 步骤条完善
3. 页面内 tooltip
4. 移动端适配

---

## 十二、非技术用户最可能卡住的地方

1. **第一次进首页不知道做什么** → 首页改造 P0
2. **看不懂模块名称** → 模块中心改造 P0
3. **分析任务不知道上传什么文件** → NewAnalysis 页面改造 P0
4. **不理解 Cron 表达式** → Schedule 改造 P1
5. **不知道"对象"是什么** → record-operations 说明 P0
6. **点了按钮报 403** → viewer 按钮隐藏 P1
7. **看到技术错误码** → 错误提示改造 P0
8. **空页面不知道下一步** → 空状态改造 P0

---

## 十三、v1.0 通过结论

**v1.0 技术功能已通过验收**，但产品体验仍有大幅提升空间。

本次 v1.1 改造**不影响 v1.0 通过结论**，仅针对可用性、易用性、引导性进行增强。

---

## 十四、改造目标

改造完成后，普通无编程经验的小团队用户：
- 打开系统 3 分钟内知道能做什么、从哪开始
- 不用看代码、不懂接口、不懂数据库，能完成主要业务流程
- 每个页面知道目的、知道下一步、知道结果怎么看
- 遇到错误知道怎么处理、联系谁
