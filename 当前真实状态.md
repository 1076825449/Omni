# 当前真实状态

本文件用于给后续 agent 一个最小、准确、可执行的仓库状态说明。

## 1. 当前阶段结论

当前仓库已经完成：

- 前端 `build`
- 前端 `lint`
- 后端基础接口测试
- 前端基础冒烟测试
- 平台 7 个正式模块的统一接入骨架

当前推荐判定为：`有条件通过`

平台骨架和 7 个模块注册已一致，7 项质量门全部通过，smoke 覆盖全部 7 个正式模块。主要剩余工作：任务/文件/日志/通知中心深化（Phase 2-5），权限体系和备份恢复（Phase 6/14），各模块细节打磨（Phase 7-13），测试扩展和文档收口（Phase 15-17）。

## 2. 已实测通过的命令

```bash
cd frontend && npm run lint
cd frontend && npm run build
cd backend && .venv/bin/pytest ../tests/backend -q
cd frontend && npm run test:e2e
python3 cli.py db status
```

## 3. 当前正式模块（共 7 个）

- `analysis-workbench`  分析工作模块
- `record-operations`   对象管理模块
- `info-query`          信息查询表
- `risk-ledger`         风险记录台账
- `learning-lab`        学习训练模块
- `dashboard-workbench` 数据仪表盘
- `schedule-workbench`  定时调度

## 4. 已完成的高价值收口

- 模块路径和 API 前缀统一到 `module key`
- 数据库、上传、备份路径统一到项目 `data/` 目录
- CLI 可从仓库根直接运行
- 首页改为真实接口驱动，不再依赖写死近期任务
- 公共页任务/文件/日志/通知基本接到真实接口
- 前端构建已完成按页面/模块拆分，当前 `build` 结果已无此前的大包告警
- `analysis-workbench` 报告导出已支持 `json/txt`
- `analysis-workbench` 已从模拟延时执行改为基于已上传文件内容的真实税务案头分析
- `analysis-workbench` 已支持围绕增值税、企业所得税、财务报表、进销项发票做交叉识别
- `analysis-workbench` 已支持识别 `有进无销`、`有销无进`、`变票`、`购销不匹配`、`白条入账`、`虚列成本`、`隐瞒收入`
- `analysis-workbench` 已支持生成面向企业的《税务事项通知书》与面向税务人员的《企业涉税风险分析报告》
- `analysis-workbench` 已支持 `XLSX` 上传解析与统一文件预览，上传页已补充税务资料模板指引
- `analysis-workbench` 上传后会返回资料类型、读取行数、字段清单和识别提醒，降低资料传错后才发现的风险
- `analysis-workbench` 已支持图片/PDF 作为佐证上传，并明确提示当前不自动 OCR
- `analysis-workbench` 已支持无表格、无图片场景下手工补录增值税、企业所得税、个人所得税关键申报指标
- `analysis-workbench` 上传后已显示标准字段确认、建议字段和缺失字段提醒
- `analysis-workbench` 结果页已生成资料缺口清单，并支持风险复核状态流转：待核实、已确认、误报、已整改、已移交
- `schedule-workbench` 的 `next_run_at` 已改为基于 cron 的可信计算，手动执行也已改为同步真实执行结果
- 分析结果页跳转到对象管理的批次筛选链路已打通
- 自动化测试已覆盖分析任务完成后的文件、日志、通知、跨模块联动与搜索回读
- 自动化测试已覆盖平台任务详情、分析任务重跑、调度执行历史、对象详情关联关系回读
- 自动化测试已覆盖 `learning-lab` 的开始练习、答题完成、收藏、统计闭环
- 自动化测试已覆盖 `dashboard-workbench` 的真实统计回读
- 前端 smoke 已覆盖全部 7 个正式模块入口
- `record-operations` 列表页的分类/状态/标签筛选与标签建议已接入真实接口，不再保留空筛选控件
- `record-operations` 的标签更新接口已从临时占位语义改为正式单对象标签更新，并带标准化去重
- `learning-lab` 的训练集与题库内容已从路由硬编码迁移为文件化配置，便于后续维护与扩展
- 任务中心已补充任务详情承接卡片，可从统一任务入口查看文件数、日志数、关联对象与来源模块
- 通知中心已补充类型筛选与回跳动作，分析完成通知可直接进入结果页
- `record-operations` 详情页已补充来源任务、关联文件、关联日志与同批对象联动
- `schedule-workbench` 已补充执行历史查看，并把手动执行结果纳入统一任务与通知链路
- `analysis-workbench` 已补充输入文件清单展示与基于原文件的重跑能力
