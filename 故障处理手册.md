# Omni 统一平台故障处理手册

## 后端启动失败

### 症状：端口被占用
```
Error: [Errno 98] Address already in use :: bind to 0.0.0.0:3000
```

**排查**：
```bash
lsof -i :3000
```

**解决**：杀掉占用进程，或改用其他端口：
```bash
.venv/bin/uvicorn app.main:app --reload --port 3001
```

### 症状：缺少依赖
```
ModuleNotFoundError: No module named 'fastapi'
```

**解决**：
```bash
cd backend
.venv/bin/pip install -r requirements.txt
```

### 症状：数据库损坏
```
sqlite3.OperationalError: database disk image is malformed
```

**解决**：从备份恢复（见恢复操作）
```bash
python3 cli.py db restore <backup_id>
```

### 症状：数据库被锁定
```
sqlite3.OperationalError: database is locked
```

**排查**：检查是否有其他进程占用 `data/omni.db`

**解决**：重启后端服务，或杀掉其他占用进程

## 前端启动失败

### 症状：端口被占用
```
Error: listen EADDRINUSE 0.0.0.0:5173
```

**排查**：
```bash
lsof -i :5173
```

**解决**：杀掉占用进程，或改用其他端口：
```bash
npm run dev -- --port 5174
```

### 症状：Node 版本不兼容
```
Error: The engine "node" is incompatible
```

**解决**：确保 Node.js ≥18：
```bash
node --version  # 确认版本
```

### 症状：依赖安装失败
```
npm ERR! network
```

**解决**：切换 npm 镜像后重试：
```bash
npm config set registry https://registry.npmmirror.com
npm install
```

## 登录失败

### 症状：登录页面不响应
**排查**：
1. 确认后端服务在 3000 端口运行
2. 确认前端已连接正确后端（检查 .env 或 VITE_API_BASE）

### 症状：用户名/密码错误
**解决**：确认账号密码正确，或重置密码：
```bash
python3 cli.py user reset-password <username> <new_password>
```

### 症状：Session 过期
**解决**：重新登录

---

## pytest 失败

### 症状：数据库被占用
```
sqlite3.OperationalError: database is locked
```

**排查**：检查是否有其他 pytest 进程或后端服务占用数据库

**解决**：关闭其他占用进程后重试

### 症状：StaticPool 相关错误
**解决**：测试使用独立临时数据库，不受生产数据影响

### 症状：测试收集失败
```
ERROR tests/backend/test_api.py - ImportError
```

**解决**：确认在正确目录运行，且虚拟环境已激活

## Playwright e2e 失败

### 症状：找不到浏览器
```
Error: Executable doesn't exist
```

**解决**：
```bash
cd frontend
npx playwright install
```

### 症状：服务未启动
```
net::ERR_CONNECTION_REFUSED
```

**解决**：确保前后端服务已启动：
```bash
# 后端
cd backend && .venv/bin/uvicorn app.main:app --reload --port 3000

# 前端
cd frontend && npm run dev
```

### 症状：测试超时
**解决**：增加超时时间，或检查服务性能：
```bash
npx playwright test --timeout 60000
```

## 备份失败

### 症状：发起备份返回 403
**原因**：当前账号没有 `platform:backup:create` 权限

**解决**：使用 admin 账号，或确认账号角色有备份权限

### 症状：备份文件不存在
**排查**：检查 `data/backups/` 目录是否存在备份 ZIP 文件

**解决**：如备份状态为 running 但文件不存在，手动重新发起备份

---

## 恢复失败

### 症状：backup_id 不存在
```
404: 备份不存在
```

**排查**：
```bash
curl http://127.0.0.1:3000/api/platform/backups
```

### 症状：备份文件已丢失
```
404: 备份文件已丢失
```

**原因**：备份 ZIP 文件被删除或路径变更

**解决**：从其他备份恢复

### 症状：恢复后数据不对
**原因**：恢复到了错误的时间点

**解决**：确认恢复前选择了正确的备份文件

---

## 权限 403

### 症状：操作返回 403 Forbidden
**原因**：当前账号没有执行该操作的权限

**排查**：确认账号角色：
- viewer 只能读，不能写
- user 不能管理系统配置（备份恢复/角色管理/模块注册）
- 只有 admin 有全部权限

**解决**：使用有权限的账号操作

### 症状：页面显示权限不足
**原因**：前端按钮权限与后端不一致（前端部分未完全按权限隐藏）

**解决**：后端已完整校验，以后端权限为准

---

## 文件预览失败

### 症状：预览显示乱码
**原因**：文件编码不是 UTF-8

**解决**：系统已对非 UTF-8 文件给出提示，或手动转换编码

### 症状：XLSX 预览失败
**原因**：文件格式不完全符合 Excel 规范

**解决**：系统已尝试解析，失败时会显示原始内容

### 症状：PDF/图片预览显示"不自动 OCR"
**原因**：系统设计如此，PDF/图片仅作为佐证文件

**解决**：这是预期行为，不算故障

---

## 文件上传失败

### 症状：上传返回 403
**原因**：当前账号没有 `platform:file:operate` 权限（viewer 角色）

**解决**：使用 user 或 admin 账号

### 症状：文件大小超限
**排查**：检查后端文件大小限制配置

---

## 日志无记录

### 症状：执行了操作但日志中心看不到
**原因**：部分操作可能尚未接入日志（已接入的操作包括：登录/任务创建/文件上传/分析执行/归档等）

**解决**：这是已知限制，日志覆盖范围已在文档说明

---

## 模块中心不显示模块

### 症状：模块中心空白
**排查**：
1. 确认后端 `/api/modules` 返回了 7 个模块
2. 确认前端路由正确注册
3. 确认数据库中模块状态为 active

**解决**：检查 seed_modules 是否正确执行：
```bash
python3 cli.py db status  # 查看模块数量
```
