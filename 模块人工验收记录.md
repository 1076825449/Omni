# 模块人工验收记录

本文件用于把“有条件通过”的剩余口径补完整，给后续 agent 和人工验收者一份可直接照着操作的记录。

## 1. analysis-workbench

- 入口位置：`/modules/analysis-workbench`
- 如何操作：
  1. 进入“新建分析”
  2. 填写名称和描述，点击“创建任务”
  3. 在上传区上传一个 `txt/csv/json/xlsx` 文件
  4. 点击“发起分析”
  5. 跳到“历史任务”后进入结果页，再进入报告页
- 成功时看到什么：历史列表出现新任务；结果页显示基于上传文件生成的摘要、文件数、相关对象数；报告页可下载 `json/txt`；结果页可跳任务中心、日志中心、对象管理模块
- 失败时看到什么：任务不存在时显示错误页；未上传文件就发起分析会返回错误提示；非法任务状态不会重复发起
- 已验证情况：自动化已覆盖创建任务、上传文件、发起运行、导出报告、通知生成、日志回读、对象联动、搜索回读；当前分析执行不再依赖固定延时模拟

## 2. record-operations

- 入口位置：`/modules/record-operations`
- 如何操作：
  1. 进入“对象列表”查看、搜索、筛选
  2. 进入“新建对象”手动创建一条记录
  3. 进入“导入数据”上传 CSV
  4. 从分析结果页点击“查看全部对象”跳入本模块，检查 `?batch=` 联动
- 成功时看到什么：列表页展示对象；新建后跳到详情页；导入后出现导入结果；分析批次跳转时自动带上批次提示；可执行归档和批量操作
- 失败时看到什么：加载失败、批量更新失败、批量删除失败、创建失败都会弹出错误提示
- 已验证情况：自动化已覆盖导入后按 `batch` 和 `q` 回读；分析结果页跳转 `?batch=` 联动已打通；前端 smoke 已覆盖模块工作台和列表入口；列表、详情、导入、新建页可形成闭环

## 3. learning-lab

- 入口位置：`/modules/learning-lab`
- 如何操作：
  1. 进入训练集列表
  2. 选择任一训练集开始练习
  3. 作答后进入结果页
  4. 再进入收藏和统计页查看数据
- 成功时看到什么：题目可提交；结果页显示答题结果；收藏页可查看已收藏题目；统计页显示练习数据
- 失败时看到什么：加载失败时页面停留当前页；提交失败时弹出错误提示
- 已验证情况：自动化已覆盖开始练习、答题完成、收藏、统计闭环；前端 smoke 已覆盖训练集入口

## 4. dashboard-workbench

- 入口位置：`/modules/dashboard-workbench`
- 如何操作：
  1. 进入模块首页
  2. 查看统计卡片、趋势、模块统计、最近活动
  3. 点击跳转按钮进入任务中心、日志中心、平台统计
- 成功时看到什么：页面显示当前用户相关的真实统计数据，并可跳到任务中心、日志中心、平台统计
- 失败时看到什么：加载失败会提示“加载仪表盘失败”
- 已验证情况：前后端前缀已统一到 `dashboard-workbench`，不再走旧 `/api/modules/dashboard`；自动化已覆盖真实统计回读；前端 smoke 已覆盖模块入口

## 5. schedule-workbench

- 入口位置：`/modules/schedule-workbench`
- 如何操作：
  1. 填写任务名称、说明、cron、任务类型并创建
  2. 在列表中执行编辑、启停、立即执行、删除
  3. 点击跳转按钮进入任务中心、日志中心、通知中心
- 成功时看到什么：任务创建后出现在列表中；编辑后字段变化可见；`next_run_at` 会根据 cron 变化；手动执行后 `last_run_at`、`last_result` 和日志会立即更新
- 失败时看到什么：非法 cron 返回错误；加载/创建/更新/执行/删除失败都有错误提示
- 已验证情况：自动化已覆盖合法/非法 cron、更新后列表回读、手动执行后的状态与日志更新；关键操作已进入统一日志
