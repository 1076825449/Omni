# Omni 统一平台管理员手册

## 角色与权限

Omni 平台有三类角色：

| 角色 | 说明 | 主要权限 |
|------|------|---------|
| admin | 管理员 | 全部权限：模块注册/角色管理/备份恢复/系统配置 |
| user | 普通用户 | 业务模块操作（任务/文件/分析/导入/执行），不能管理系统配置 |
| viewer | 访客 | 只读：查看列表、详情、统计，不能新增/编辑/删除/导入/执行 |

## 用户管理

### 创建用户

```bash
python3 cli.py user create <username> <password> --role <role>
```

示例：
```bash
python3 cli.py user create viewer viewer123 --role viewer
python3 cli.py user create testuser user123 --role user
```

### 重置密码

```bash
python3 cli.py user reset-password <username> <new_password>
```

### 查看用户列表

```bash
python3 cli.py user list
```

### 查看数据库状态

```bash
python3 cli.py db status
```

## 模块管理

### 查看已注册模块

前端：模块中心页面
后端：`GET /api/modules`

7 个正式模块：
- `analysis-workbench` — 分析工作模块
- `record-operations` — 对象管理模块
- `info-query` — 信息查询表
- `risk-ledger` — 风险记录台账
- `learning-lab` — 学习训练模块
- `dashboard-workbench` — 数据仪表盘
- `schedule-workbench` — 定时调度

### 注册新模块（admin）

```bash
curl -X POST http://127.0.0.1:3000/api/modules/register \
  -H "Content-Type: application/json" \
  -b "omni_session=<admin_session>" \
  -d '{"key": "new-module", "name": "新模块", "type": "list"}'
```

## 备份与恢复

### 发起备份

前端：设置 → 备份中心 → 填写备份名称 → 发起备份

API：
```bash
curl -X POST http://127.0.0.1:3000/api/platform/backup \
  -H "Content-Type: application/json" \
  -b "omni_session=<admin_session>" \
  -d '{"name": "Phase1完成备份", "note": ""}'
```

### 查看备份记录

```bash
curl http://127.0.0.1:3000/api/platform/backups \
  -b "omni_session=<admin_session>"
```

### 下载备份

从备份记录点击「下载」按钮，或：
```bash
curl -O http://127.0.0.1:3000/api/platform/backups/<backup_id>/download \
  -b "omni_session=<admin_session>"
```

### 恢复数据库

**重要：恢复会覆盖当前数据，请在恢复前确认备份完整性，并停止服务。**

```bash
# 1. 停止后端服务

# 2. 执行恢复
python3 cli.py db restore <backup_id>

# 3. 重启后端服务
cd backend
.venv/bin/uvicorn app.main:app --reload --port 3000
```

恢复过程会自动：
- 验证备份文件完整性（ZIP + SQLite header）
- 创建安全备份 `pre-restore-{timestamp}.db`
- 提取 `omni.db` 和 `uploads/` 目录
- 记录操作日志

## 角色权限管理

前端：设置 → 角色管理（admin 专属）

可查看和编辑的角色权限包括：
- 模块访问权限（view/operate/export）
- 平台公共权限（task/view&operate、file/view&operate、log/view&export、backup、settings、role:manage）

**注意**： viewer 角色默认只读，不能进行任何写操作。user 角色可使用业务模块，但不能管理系统配置。

## 日志审计

入口：日志中心

可查看：
- 所有平台操作日志（登录/任务创建/文件上传/分析执行/导入/归档等）
- 按操作类型、模块、结果（成功/失败）筛选
- 失败日志有醒目标识

管理员可通过日志追踪：
- 用户登录行为
- 备份创建/恢复操作
- 权限变更
- 模块注册/配置变更

## 常见维护任务

```bash
python3 cli.py server status
```

```bash
python3 cli.py server restart
```

### 清理过期会话

```bash
python3 cli.py session cleanup
```

### 数据库初始化（慎用）

```bash
python3 cli.py db init
```

## 安全建议

1. **修改默认密码**：立即将 admin/admin123 密码改为强密码
2. **定期备份**：建议每周至少一次备份
3. **慎用 viewer 共享**：viewer 仅供需要只读权限的人员
4. **备份文件保管**：备份 ZIP 包含完整数据，妥善保管
5. **恢复前确认**：执行恢复前确认备份来源可靠
