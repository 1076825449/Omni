# Omni 统一平台部署与启动说明

## 环境要求

- **Node.js**: ≥18
- **Python**: 3.9+
- **操作系统**: macOS / Linux（Windows 通过 WSL2 或 Git Bash）
- **数据库**: SQLite（随项目存储在 `data/omni.db`）

## 快速启动

### 1. 克隆项目

```bash
git clone git@github.com:1076825449/Omni.git
cd Omni
```

### 2. 后端启动

```bash
cd backend

# 创建虚拟环境（如尚未创建）
python3 -m venv .venv

# 安装依赖
.venv/bin/pip install -r requirements.txt

# 初始化数据库（如尚未初始化）
cd ..
python3 cli.py db init

# 启动后端服务
cd backend
.venv/bin/uvicorn app.main:app --reload --port 3000
```

后端地址: http://127.0.0.1:3000

### 3. 前端启动（新终端）

```bash
cd Omni/frontend
npm install
npm run dev
```

前端地址: http://127.0.0.1:5173

### 4. 登录

默认账号: `admin` / `admin123`

## 项目结构

```
Omni/
├── backend/
│   ├── app/
│   │   ├── main.py          # FastAPI 入口

│   │   └── models/          # 数据库模型

│   └── .venv/               # Python 虚拟环境
├── frontend/
│   ├── src/
│   │   ├── pages/           # 平台公共页面（任务中心/文件中心/日志中心/通知中心等）
│   │   ├── modules/         # 7个正式模块前端
│   │   └── services/        # API 客户端
│   └── package.json
├── data/                    # 数据库、上传文件、备份（本地存储，不提交到 Git）

└── README.md
```

## 数据初始化

```bash
# 查看当前数据库状态
python3 cli.py db status

# 创建额外用户
python3 cli.py user create viewer viewer123 --role viewer
python3 cli.py user create testuser user123 --role user

# 列出所有用户
python3 cli.py user list
```

## 备份

```bash
# 发起备份（需要 admin 权限）
curl -X POST http://127.0.0.1:3000/api/platform/backup \
  -H "Content-Type: application/json" \
  -b "omni_session=<your_session_cookie>" \
  -d '{"name": "手动备份", "note": ""}'

# 查看备份记录
curl http://127.0.0.1:3000/api/platform/backups \
  -b "omni_session=<your_session_cookie>"

# 下载备份
curl -O http://127.0.0.1:3000/api/platform/backups/<backup_id>/download \
  -b "omni_session=<your_session_cookie>"
```

## 恢复

```bash
# 1. 停止后端服务

# 2. 执行恢复（需 admin 权限）
python3 cli.py db restore <backup_id>

# 3. 重启后端服务
```

## 常见问题

- **后端端口被占用**: `lsof -i :3000` 查看进程
- **前端端口被占用**: `lsof -i :5173` 查看进程
- **数据库锁定**: 重启后端服务或检查是否有其他进程占用 `data/omni.db`
